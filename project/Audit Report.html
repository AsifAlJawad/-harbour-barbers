Backend Integration Checklist

Recommended Build Order
1
Business & staff records. Seed businesses (Delta, BC, info@harbourbarbers.ca) and staff (Marco, James, Sofia). Data is now consistent across all files.
2
Services table. 11 canonical services with keys matching hb_prices localStorage schema. The Owner Pricing tab already writes this exact key structure — wire its Save to a PATCH /services/prices endpoint.
3
Availability engine. Slots from Mon–Wed & Fri 9–6, Thu 9–7, Sat 10–5, Sun closed. The dynamic date selects in Harbour Barbers and Customer Portal already filter by this schedule — replace with GET /availability?date=…&barber=….
4
Auth + bookings. Single auth through Customer Portal. Booking confirmation redirect from main site modal is wired. Add JWT or session cookie on login, then POST /bookings on Confirm.
5
Loyalty transactions. 1 pt per $1. POST /loyalty/earn on booking completion. Tier thresholds: Bronze 0–199, Gold 200–499, Platinum 500+. Customer Portal Rewards view is already wired to show these tiers.
6
Walk-in endpoint. Staff Dashboard walk-in form → POST /bookings with source: "walk_in", status: "active", time: now. No date picker needed — slot is always "now".
7
Drag-reorder endpoint. PATCH /schedule/reorder with ordered appointment IDs. Update a display_order field only — never change the appointment time.
8
CMS for Site Tweaks + Settings. Contact, hours, gallery, brand colour, headings — all editable in Owner Dashboard Settings tabs. On save, POST /cms/settings. Build after core booking flow is stable.
